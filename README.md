This repository documents the foundational "Data Engineering" phase of the RAG pipeline. High-performance LLM applications are only as powerful as the data they can retrieve; these notebooks demonstrate the rigorous preprocessing required to transform raw, unstructured documents into a queryable knowledge base. Drawing from experience in processing over 10 million metadata records from large-scale clinical databases, this layer establishes the "Single Source of Truth" necessary for any context - aware neural network or LLM. By utilizing LangChain’s specialized loaders in document_loading.ipynb, the pipeline ensures that essential metadata is preserved, allowing for the same high-precision keyword extraction and 30% noise reduction achieved in prior NLP research using TF-IDF.


The second phase, detailed in document_splitting.ipynb, addresses the complex challenge of semantic chunking to optimize model input quality. Based on expertise in LSTM temporal dependencies and BERT implementations, this notebook explores recursive and token-based splitting to maintain logical continuity. By carefully calibrating chunk sizes and contextual overlaps, the workflow minimizes the risk of losing critical information at the boundaries — a technical necessity when identifying important features to boost model predictability, as seen in clinical time-series data where accuracy was improved from 60% to 88%. This repository serves as an experimental sandbox to benchmark how different splitting strategies impact the ultimate accuracy and efficiency of the retrieval process.
